# 진행자 가이드

## 준비물

- `participant-worksheet.md`를 참가자 수만큼 복사한다.
- 참가자에게 미리 요청한다: "만들고 싶은 것, 자동화하고 싶은 것, 정리하고 싶은 문제를 하나 가져오세요."
- 특정 AI 도구 구독은 요구하지 않는다.

## 시작 전 확인

각 참가자에게 아래 4가지만 확인한다.

1. 사용하는 OS: Windows, macOS, Linux, iPad, 웹 전용 등
2. 사용할 AI: ChatGPT, Claude, Gemini, Cursor, Windsurf, Replit, Bolt, Lovable, Claude Code, Codex 등
3. 로컬 개발 가능 여부
4. 오늘 다루면 안 되는 개인정보나 업무 비밀 여부

## 목표 축소 방법

참가자의 목표가 크면 아래 문장으로 줄인다.

```text
오늘은 전체 완성이 아니라, 이 목표에서 가장 작은 첫 결과물 하나만 만든다.
클릭 1번, 입력 1개, 변환 1회, 초안 1개, 비교표 1개 중 하나로 줄이자.
```

예시:

- "쇼핑몰 만들기" -> "상품 3개를 보여주는 첫 화면 만들기"
- "회사 업무 자동화" -> "샘플 파일 1개를 원하는 형식으로 바꾸기"
- "유튜브 채널 운영" -> "영상 아이디어 5개와 검토 기준 만들기"
- "데이터 분석" -> "샘플 20행을 분류 규칙대로 정리하기"

## 공통 프롬프트 템플릿

참가자는 작업 시작 전에 아래 템플릿을 자기 목표에 맞게 채운다.

```text
내 목표:

사용자:

입력:

출력:

데이터:

제약:

예외 상황:

오늘 만들 가장 작은 첫 결과물:

너는 바로 구현하지 말고, 먼저 작업을 3단계로 나누고 빠진 정보가 있으면 질문해줘.
```

## 구조 설명 프롬프트

첫 결과물이 나온 뒤 사용한다.

```text
방금 만든 결과물의 구조를 초보자가 이해할 수 있게 설명해줘.
특히 다음을 알려줘.

1. 핵심 구성 요소
2. 입력이 처리되는 흐름
3. 내가 수정하면 위험한 부분
4. 내가 안전하게 바꿔볼 수 있는 부분
5. 아직 검증하지 않은 부분
```

## 컨텍스트 재시작 프롬프트

대화가 길어지거나 AI가 엉뚱한 답을 할 때 사용한다.

```text
새 대화에서 이어갈 수 있도록 현재 작업을 요약해줘.

형식:
- 목표:
- 지금까지 완료:
- 현재 산출물:
- 중요한 결정:
- 아직 안 한 것:
- 다음 작업:
- 주의할 점:
- 절대 바꾸면 안 되는 것:
```

새 대화에는 위 요약과 다음 요청 하나만 넣게 한다.

## 디버깅 프롬프트

에러 메시지만 반복해서 붙여넣지 않게 한다.

```text
문제가 생겼다. 바로 전체를 고치지 말고 원인을 좁혀줘.

마지막으로 정상 동작한 상태:
방금 바꾼 것:
증상:
예상한 동작:
실제 동작:
내가 의심하는 범위:

먼저 가능한 원인 3개를 말하고, 확인 순서를 제안해줘.
```

## 버전 저장 안내

코드 프로젝트:

```text
변경 전 정상 상태를 저장한다.
가능하면 git commit을 사용한다.
```

웹 도구/문서/자동화:

```text
변경 전 복제본, 버전 기록, 스크린샷, 중간 결과 복사 중 하나를 남긴다.
되돌아갈 수 있는 지점이 있으면 충분하다.
```

## 보안 점검 질문

마지막 10분에 모두에게 묻는다.

- 개인정보를 AI에 넣었는가?
- API 키, 비밀번호, 토큰을 코드나 문서에 직접 적었는가?
- 공개 링크나 공유 권한이 열려 있는가?
- 로그인, 결제, 고객 데이터가 포함되는가?
- 실제 배포 전에 더 확인해야 할 위험은 무엇인가?

## 진행자 개입 기준

- 설치가 10분 이상 막히면 웹 기반 경로로 바꾼다.
- 목표가 너무 크면 최소 산출물 하나로 줄인다.
- AI 결과를 이해하지 못하면 구현을 멈추고 구조 설명 단계로 돌아간다.
- 참가자가 "그냥 돌아가면 됐다"고 하면 검증 질문을 최소 2개 쓰게 한다.
- 보안 위험이 보이면 결과물 완성보다 위험 제거를 우선한다.

## 마무리 회고 질문

각자 1분씩 답한다.

1. 오늘 AI에게 가장 잘 준 컨텍스트는 무엇인가?
2. AI 결과를 검증한 방법은 무엇인가?
3. 다음에 같은 목표를 계속한다면 첫 요청을 어떻게 바꿀 것인가?
